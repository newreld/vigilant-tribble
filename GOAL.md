# GOAL & CONCEPT

This is the canonical, living statement of *what this project is trying to be
and why*. Keep it updated as the vision evolves. It is intentionally separate
from the [README](README.md) — the README is for **players** (what the game is
and how to run it); this document is for **us** (the goal and the concept).

For the deep research, the psychology, the design levers, and the full
iteration/decision log, see [DESIGN.md](DESIGN.md). This file is the short,
high-level "north star."

---

## The goal

Build a genuinely good, **return-to-it** game with **top-100 App Store
potential as a solo build** — and do it *honestly*.

It is the **"too good to be true"** version of the over-advertised
hyper-casual genre: it actually delivers the satisfying thing those ads
overpromise, and there is **no catch**.

> **No ads. No in-app purchases. No energy/lives timers. No loot-box gambling.
> No catch. Just merge.**

Every reward is earnable by play, instantly, for free. The core loop has to be
fun enough that someone comes back tomorrow *without* the joke — the parody is
differentiation and marketing, never the reason to keep playing.

## The concept

A physics **merge game** (the *Suika / watermelon-game* lineage), themed in
space: drop and merge celestial bodies — **asteroid → comet → moon → planet →
ringed planet → dwarf star → star → galaxy → black hole** — and merge two black
holes to trigger a **BIG BANG** that clears the board for a huge bonus.

On top of that real game sits a light **persiflage** (loving parody) of the
hyper-casual genre's dark patterns: the game is "rigged in the player's favor,"
winking at the manipulations instead of using them. The game comes first; the
joke is seasoning.

## Why this shape (the short version)

- **Accessible + deep** — "understand in seconds, master in hours." Real skill
  and planning, not autopilot.
- **Loop, not content treadmill** — procedurally endless; no hundreds of
  hand-authored levels to keep it fresh.
- **Organic virality** — Suika Game blew up on streams with *zero* ad spend,
  which is the only realistic path to the charts without a marketing budget,
  and it fits the honest thesis: it spreads because it's good.
- **Honest by construction** — turn-paced, no clock, no energy, no IAP. The
  only randomness is the fair next-piece variety (like Tetris); a win is always
  *built*, never handed to you, and the tests assert this.

## The open question we're still chasing

Is the core loop fun enough on its own that someone returns tomorrow **without**
the joke? Validating (and, if needed, deepening) that loop is the real work. The
instrumentation and iteration log live in [DESIGN.md](DESIGN.md).
